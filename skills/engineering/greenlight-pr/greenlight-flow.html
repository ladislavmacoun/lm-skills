Core

Triage Process

Phase 1
Evaluate
  • Sub-agent per comment
  • Read code context
  • FIX / DISAGREE / DEFER
  • Confidence: H / M / L
Phase 2
Synthesize
  • Collect verdicts
  • Answer in context?
  • Surface decisions
  • Quality-check fixes
Phase 3
Act
  • Fix (one commit)
  • Reply to all comments
  • Post summary
  • Push → wait → loop
Per-Comment Decision
for each comment:
  ├─ correct?        no → DISAGREE // explain
  ├─ actionable?    no → DEFER // out of scope
  ├─ workaround?   yes → DISAGREE // real fix
  ├─ complex?      yes → DISAGREE // simpler
  ├─ needs user?    yes → ASK // batch Qs
  └─ else → FIX // simplest change
push → re-snapshot ↺
push → CI → wait review → re-snapshot ↺
